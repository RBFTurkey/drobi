# RBF Tutor — раздел «Дроби»

Интерактивный учебный сайт по дробям: 11 тем, в каждой **урок** и **проверка знаний** с бесконечной генерацией заданий, генератор печатных наборов и сохранение результатов (локально + опциональный журнал в Google Sheets).

Весь сайт — это один файл `index.html`. Никаких сборок и зависимостей: достаточно открыть его в браузере или выложить на GitHub Pages.

---

## 1. Деплой на GitHub Pages

1. Зайди на **github.com** → **New repository**. Назови, например, `drobi`. Сделай его **Public**. Нажми **Create**.
2. На странице репозитория: **Add file → Upload files**. Перетащи `index.html` (можно и `README.md`, `apps_script.gs`). Нажми **Commit changes**.
3. Открой **Settings → Pages**.
4. В разделе **Build and deployment → Source** выбери **Deploy from a branch**.
5. **Branch:** `main`, папка `/ (root)`. Нажми **Save**.
6. Подожди ~1 минуту и обнови страницу. Появится ссылка вида:

   ```
   https://ТВОЙ_ЛОГИН.github.io/drobi/
   ```

Эту ссылку можно отправлять ученикам. Сайт работает на телефоне, планшете и компьютере.

> Обновление сайта: загрузи новый `index.html` поверх старого (Add file → Upload files → Commit). Pages обновится автоматически за минуту.

## 2. Журнал результатов в Google Sheets (необязательно)

Без этого шага результаты всё равно сохраняются на устройстве ученика и выгружаются в CSV из раздела «Обзор». Журнал нужен, чтобы **видеть результаты всех учеников удалённо, в одной таблице**.

1. Создай новую таблицу **Google Sheets**.
2. **Расширения → Apps Script**.
3. Удали пустой код, вставь содержимое файла **`apps_script.gs`**, сохрани (Ctrl+S).
4. **Развернуть → Новое развёртывание**. Тип — **Веб-приложение**.
   - **Запуск от имени:** Я
   - **У кого есть доступ:** Все
   - **Развернуть**. Разреши доступ, если попросит.
5. Скопируй **URL веб-приложения** (заканчивается на `/exec`).
6. Открой сайт → в меню слева **«Настройки и журнал»** → вставь URL → **Сохранить ссылку** → **Отправить тест**.
7. В таблице появится строка — журнал работает.

После этого на каждой проверке кнопка **«Сохранить результат в журнал»** дописывает в таблицу строку: дата, ученик, тема, верно, попыток, точность.

> Технический момент: отправка идёт «вслепую» (fire-and-forget), поэтому сайт не показывает ответ сервера — проверяй факт записи в самой таблице.

> Где хранится ссылка на журнал: в браузере того устройства, где ты её ввёл. Если хочешь «зашить» её для всех — впиши свой URL в `index.html` в строку `let tutorEndpoint='';` (внутри `const STORE...`), например `let tutorEndpoint='https://script.google.com/.../exec';`.

## 3. Что внутри каждой темы

- **Урок** — короткое объяснение простым языком (для 10–12 лет) с визуальными моделями: полоски, числовая прямая, модель площади, сотенный квадрат.
- **Проверка знаний** — бесконечный поток заданий с мгновенной проверкой, подсказками и разбором ошибок.
- **Сгенерировать набор** — создаёт лист со свежими заданиями (6/10/15/20) и ответами; кнопка «Печать / PDF».

## 4. Сохранение и сброс

- Имя ученика и результаты хранятся на устройстве (localStorage).
- «Обзор и результаты»: таблица по темам, **CSV** и **копирование для Google Sheets**, кнопка **Сброс**.
- При открытии в режиме предпросмотра (не как обычная страница) localStorage может быть недоступен — тогда данные живут только в текущей сессии.
